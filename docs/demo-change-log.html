"快照记录"升级为"变更记录"，转账自动记录余额变化

改动说明

1. 转账立即更新余额 新增
转账提交后，源账户 current_balance 自动减少，目标账户自动增加

2. "快照记录"→"变更记录" 改进
表格标题改名，新增"来源"列，区分 📸快照 和 🔄转账

3. 转账自动插入变更记录 新增
转账时自动生成两条变更记录（一出一入），和快照记录共存

✅ 转账金额精确，理论变化已包含转账，预计余额 = 实际余额，差额为 0。无需额外处理。

技术变更

AccountBalance 模型新增字段：
• source — VARCHAR(20)，默认 'snapshot'，转账时设为 'transfer'

转账逻辑变更：
• 提交转账时，除了生成 Transaction 记录外，同时：
  ① 更新源账户 current_balance -= 金额
  ② 更新目标账户 current_balance += 金额
  ③ 插入源账户 AccountBalance（source='transfer'，变更金额为负）
  ④ 插入目标账户 AccountBalance（source='transfer'，变更金额为正）

页面变更：
• 表格标题：快照记录 → 变更记录
• 新增"来源"列：📸 快照 / 🔄 转账
• 转账行浅蓝底色区分
• 按时间倒序排列（最新的在上面）
